01THESIS

Three independent wins.

The current product wastes effort on three orthogonal axes. Fix any one in isolation; fix all three and the product becomes materially cheaper, faster, and easier to operate.

01
Platform leverage

Stop reinventing what Telegram now ships.

Bot API has matured a lot since the product launched: join-request approval (5.4), Stars subscriptions (8.0), self-enforcing channel subscriptions (8.1), Mini Apps with HMAC auth, and native chat/user pickers. Six pieces of custom code can simply go.

~30% code surface retired
02
React, don't poll

Daily Purge → event-driven re-verify.

Rescanning every member's wallet every day is O(chats × members × tokens × chains) wasted work — almost everyone's balance is unchanged. Webhooks fire only when a relevant transfer happens.

~−86% RPC bill · 720× faster eviction
03
Free signatures

Self-transfer dust → message signatures.

Self-transfer is clever (link-share immune, smart-wallet friendly) but charges users $0.50–$5 in gas on Ethereum L1. SIWE / SIWS / ton_proof do the same job for $0 with explicit replay semantics.

$0 verification cost to user

04DIFF

What changed.

Every line of v1 code falls into one of three buckets: deleted (Telegram or events do it now), upgraded (kept but improved), or new (didn't exist before).

Removed

deleted from v2 / Telegram or events do this now

  • Daily Purge cronRescanning every member every 24h. Replaced by event triggers + weekly belt-and-braces sweep.
  • Ban-after-join hot pathbanChatMember + unbanChatMember loop. Non-holders never get in to begin with.
  • 10-step DM wizardLinear FSM where one wrong tap restarts the chain. Replaced with a Mini App form.
  • Self-transfer dust watcherPer-chain watcher matching (from==to==addr, value==nonce). Replaced by free signatures.
  • @username-keyed whitelistBroke when users renamed or had no public username. Replaced by numeric tg_user_id.
  • Custom payment receipt codeOne-time SOL fee logic. Telegram Stars handles billing for paid tiers.

Upgraded

kept but improved / hardened against gaps

  • Auth Verifier (was: Wallet Link Service)Polymorphic dispatch on method column — SIWE / SIWS / ton_proof / dust as legacy. Smart-contract wallets via EIP-1271 / 6492.
  • Net-Worth PricerMulti-source: CoinGecko primary → on-chain TWAP fallback → Multichain Metrics for cross-chain unification only. Stale-mode flag.
  • Settings Store + Mini AppReal form UI instead of message-driven FSM. Whitelist on numeric IDs. Auth via initData HMAC.
  • Holdings AggregatorSame role, way colder path — caches (chain, address, token) in Redis 5–15 min. Read on event, not on cron.
  • Per-user one-shot invitescreateChatInviteLink(member_limit=1, expire=now+1h). Telegram audits per-link attribution for free.

Added

new in v2 / capabilities that didn't exist before

  • Join Request HandlerSubscribes to chat_join_request; runs gate predicate; calls approveChatJoinRequest or declineChatJoinRequest.
  • Re-verify Worker (event-driven)Consumes Alchemy + Helius webhooks; idempotent on (chain, txhash, logIndex); reorg buffer; weekly fallback sweep.
  • Webhook subscription layerAlchemy Address Activity for EVM (100k addrs/webhook) + Helius Enhanced Webhooks for Solana.
  • EIP-1271 / 6492 verifierSmart-contract wallets (Safe, Argent, Coinbase Smart Wallet) work for free with signatures.
  • Stars subscription pathcreateInvoiceLink(currency='XTR', subscription_period=2592000). Telegram self-renews; auto-removes lapsed users on channels.
  • Mini App settings frontendVite + React on Vercel · authed by initData HMAC · serves both owner config and end-user verify.

06MIGRATION

Four weeks.

Single dev. Each week is independently shippable; each previous week's work keeps running until the next is shadow-validated. Nothing flips on a single day.

WK01

Switch to approve-before-join

Flip every gated chat to invite links with creates_join_request: true. Subscribe to chat_join_request in allowed_updates. Run the gate predicate, then approve or decline. The old ban-after-join code path becomes dead code.

LOW · ADDITIVE
WK02

Wire up webhooks in shadow

Provision Alchemy Address Activity for EVM + Helius Enhanced Webhooks for Solana. Route to a "potentially-disqualifying" queue. Daily Purge keeps running — webhook decisions are logged but not enforced yet. Compare the two outputs for 7 days.

LOW · SHADOW MODE
WK03

Build the Mini App

Vite + React on Vercel. Auth via initData HMAC. Use KeyboardButtonRequestChat to pick the chat to gate, KeyboardButtonRequestUsers for whitelist. Migrate whitelist storage to numeric tg_user_id. The DM wizard stays as fallback for old clients.

MID · NEW SURFACE
WK04

Ship free signatures + retire Purge

Add SIWE / SIWS / ton_proof verifiers behind a method column. Self-transfer dust stays as fallback. Once webhook shadow has matched daily Purge for 7 days, retire the daily cron in favor of weekly belt-and-braces. Monitor for 2 weeks.
